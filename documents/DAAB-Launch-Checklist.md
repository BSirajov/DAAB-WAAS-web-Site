# DAAB bilingual site — launch checklist

Use this before pushing to GitHub Pages or updating public links.

## One-command prep

```bash
python helpers/_build_bilingual_tree.py
python helpers/_publish_en_pages.py all
python helpers/_validate_bilingual.py
```

All three should succeed with no errors.

## Local smoke test (http://localhost:8010)

1. Open `/` — should land on `/az/` (or show gateway with `?choose=1`).
2. On any `/az/` page, click **EN** in the nav — same page in English.
3. Click **AZ** — back to Azerbaijani.
4. Open each English page from `/en/index.html` cards.
5. Scientists: list filters, profiles search, CV links if any.

## Production URLs to link publicly

| Audience | URL |
|----------|-----|
| Default (Azerbaijani) | `https://daab-waas.com/az/` |
| English | `https://daab-waas.com/en/` |
| Language choice | `https://daab-waas.com/` |

Update **Google Sites** and email signatures to use `/az/` or `/en/`, not old `*_az.html` filenames.

## SEO files (generated by build)

- `sitemap.xml` — all `/az/` and `/en/` pages with hreflang alternates
- `robots.txt` — points to sitemap
- Per-page `canonical` + `hreflang` — injected by `daab-shell.js` when `i18n/routes.json` loads

## After content edits

| You changed | Run |
|-------------|-----|
| Legacy `*_az.html` or copy in `az/` | `_build_bilingual_tree.py` then `_publish_en_pages.py <page>` |
| Only English strings in publish maps | `_publish_en_pages.py <page>` or `all` |

## Known partial English (not blockers for launch)

- Charter: article bodies in Azerbaijani
- Activities: full event texts in Azerbaijani
- Profiles: biography paragraphs in Azerbaijani

Banners on those EN pages explain where to read full text.
